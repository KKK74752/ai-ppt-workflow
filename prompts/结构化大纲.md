# 结构化大纲生成

## 解决的问题

客户提供长文本（述职报告、论文摘要、产品介绍），需要转化为 PPT 内容框架。

传统做法：人工阅读 → 手动分段 → 提炼标题 → 调整逻辑，耗时 20-30 分钟/次。

## Prompt

```
请你帮我完成PPT页面中文字结构逻辑的梳理，并总结提炼结论型小标题。

【结构梳理】
根据文本信息，按照不同的信息主题，区分文段的结构。分为N个版块。
将文本内容按信息关联性重组。
要求：
1. 无关信息单独分组
2. 用缩进体现层级关系
3. 保留原文信息，适当删除连接词、形容词

【标题提炼】
为每个内容模块提炼结论型标题，要求：
1. 控制在12字以内
2. 包含关键数据或核心结论
3. 使用"动词+结果"结构

【总标题】
基于所有模块核心结论，提炼一个18字内的总结性标题：
1. 使用"结论先行+价值点"结构
2. 内容精简但总体意思不变

【原始文本】
{客户提供的文字内容}
```

## 效果

| 指标 | 手工 | AI 辅助 |
|------|------|---------|
| 耗时 | 20-30min | < 2min |
| 结构一致性 | 依赖个人状态 | 模板化输出 |
| 迭代成本 | 改需求=重做 | 重新生成 |

## 适用边界

- ? 长文本类内容（述职/论文/报告）
- ? 纯数据类内容（用图表方案替代）
- ? 客户已有明确大纲（直接用）
