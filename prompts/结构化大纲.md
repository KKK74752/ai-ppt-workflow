# 结构化大纲生成

## 场景

客户交付长文本（述职报告、论文、商业计划书），需转化为 PPT 逻辑框架。

## Prompt

```
请你帮我完成PPT页面中文字结构逻辑的梳理，并总结提炼结论型小标题。

【结构梳理】
首先，你需要根据文本信息，按照不同的信息主题，区分文段的结构。
比如分为N个版块，分别是：段落1、段落2、段落3……
其次，将文本内容按信息关联性重组。
要求：
1. 无关信息单独分组
2. 用缩进体现层级关系
3. 务必保留原文信息，确保观众阅读时能够理解文段信息，但可适当删除连接词、形容词等无关信息

【标题提炼】
请为每个内容模块提炼结论型标题，要求：
1. 控制在12字以内
2. 包含关键数据或核心结论
3. 使用"动词+结果"结构（如：市场规模突破X亿、用户增长超预期）

【总标题】
基于本页所有模块的核心结论，提炼一个18字内的总结性标题，要求：
1. 使用"结论先行+价值点"结构（如：用户留存率提升30%，实现精细化运营突破）
2. 内容稍微精简但是总体需要呈现的意思不能变

【原始文本】
{客户提供的文字内容}
```

## 产出示例

输入：一段 800 字的述职报告原文

输出：
```
总标题：完成3个核心项目交付，团队效率提升25%

段落1：项目交付成果
  - 完成X项目上线，覆盖Y用户
  → 标题：核心项目按时交付率达100%

段落2：流程优化
  - 引入自动化测试，减少回归时间
  → 标题：测试效率提升40%

段落3：团队建设
  - 培养2名新人独立承担模块
  → 标题：新人独立交付周期缩短至2周
```
